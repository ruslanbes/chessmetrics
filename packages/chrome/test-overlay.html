🎯 Chessmetrics Overlay System Test

📋 Testing Instructions

  1. Install the userscript in Tampermonkey/Greasemonkey
  2. Go to a Lichess analysis page with a position containing pinned pieces
  3. Open browser console (F12 → Console tab)
  4. Run the commands below to test the overlay system
  5. Navigate through moves to see pin indicators (📍) appear on pinned pieces

🔧 Setup Commands

Copy and paste these commands in the browser console on a Lichess page:

// Enable debug mode to see detailed logs window.chessmetrics.setDebugMode(true)
// Start move tracking (this will also start the overlay system) window.chessmetrics.startMoveTracking()

🎯 Test Commands

Use these commands to test the overlay system:

// Check if move tracking is active window.chessmetrics.isMoveTrackingActive()
// Check if debug mode is enabled window.chessmetrics.isDebugMode()
// Stop move tracking (this will also stop the overlay) window.chessmetrics.stopMoveTracking()

🔍 What to Look For

  • Console logs showing overlay engine initialization
  • Pin indicators (📍) appearing in the bottom-right corner of pinned piece squares
  • Indicators updating when you navigate to different moves
  • Indicators disappearing when you stop move tracking

🧪 Test Positions

Try these Lichess analysis URLs with known pinned pieces:

  • Lichess Analysis Board - Set up a position with pinned pieces
  • Look for positions where a piece is pinned to the king by a bishop, rook, or queen

🐛 Troubleshooting

If the overlay doesn't work:

  • Check browser console for error messages
  • Ensure you're on a Lichess page with a chess board
  • Try refreshing the page and running the commands again
  • Make sure the userscript is properly installed in Tampermonkey

📊 Expected Console Output

When working correctly, you should see logs like:

OverlayEngine: Initialized with Lichess (Chessground) adapter OverlayEngine: Registered visualizer: pinned-pieces 🎯 MoveTracker: Position changed to ply 10 OverlayEngine: Rendered 2 overlays
